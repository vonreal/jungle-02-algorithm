# 알고리즘 커밋 컨벤션

> Angular Commit Convention 기반, 알고리즘 문제 풀이에 최적화

## 형식

```
<type>(<scope>): <subject>

<body> (선택)
```

---

## Type

| type | 설명 | 사용 시점 |
|------|------|-----------|
| `solve` | 문제 풀이 완료 | AC(정답) 받았을 때 |
| `attempt` | 풀이 시도 | 시간초과, 오답, 미완성이지만 시도 기록을 남길 때 |
| `refactor` | 풀이 개선 | AC 후 시간복잡도 개선, 코드 정리 |
| `feat` | 기능 구현 | basic 빈 코드 채우기, 자료구조 직접 구현 |
| `docs` | 문서 작성 | README, 풀이 해설, 개념 정리 |
| `fix` | 버그 수정 | 기존 풀이의 엣지케이스 수정 |
| `chore` | 기타 | 폴더 구조 변경, 설정 파일 등 |

---

## Scope (개념 태그)

이번 주차 기준:

| scope | 설명 |
|-------|------|
| `tree` | 트리, 이진 트리, BST |
| `graph` | 그래프 표현, 인접 리스트/행렬 |
| `bfs` | 너비 우선 탐색 |
| `dfs` | 깊이 우선 탐색 |
| `topological-sort` | 위상 정렬 |

---

## Subject 작성 규칙

- 문제 풀이: `플랫폼 문제번호 문제이름`
- basic: `구현 내용`
- 한글 사용 OK
- 끝에 마침표 없이

---

## 예시

### 문제 풀이 완료
```
solve(tree): BOJ 9372 상근이의 여행
```

### 시도했지만 미해결
```
attempt(bfs): BOJ 2293 동전 2

- BFS로 접근했으나 시간초과
- DP로 전환 필요할 듯
```

### 풀이 개선
```
refactor(dfs): BOJ 1707 이분 그래프

- 재귀 DFS → 스택 DFS로 변경
- 메모리 사용량 개선
```

### basic 구현
```
feat(bfs): 너비 우선 탐색 구현

- 큐 기반 BFS 구현
- 방문 체크 로직 포함
```

### 개념 정리
```
docs(tree): 트리 순회 개념 정리
```

### 코어타임 코딩테스트
```
solve(tree): LC Maximum Depth of Binary Tree

- 코어타임 모의 코테
```

### 시간초과 후 재도전 성공
```
solve(bfs): BOJ 2293 동전 2

- 이전 BFS 접근에서 DP로 전환
- O(n*k) 풀이
```

## body 작성 팁 (선택사항)

`attempt`일 때는 body에 아래 내용을 남기면 복기할 때 유용함:
- 어떤 접근법을 시도했는지
- 어디서 막혔는지
- 다음에 시도할 방향

`solve`일 때 골드 이상이면:
- 핵심 아이디어 한 줄
- 시간복잡도
