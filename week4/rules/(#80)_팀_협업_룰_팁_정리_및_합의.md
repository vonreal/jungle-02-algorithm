# 금주차 팀 협업 룰/팁 정리

## 스크럼 미팅
- 매일 오전 10시 출근 후 스크럼 미팅 진행
- 각자 당일 할 일을 간단히 공유 (GitHub Projects 기준으로)
- 형식 예시: "어제 한 것 → 오늘 할 것 → 막힌 것" 3줄 공유

## 코어타임 운영

### 방향
- 이번 주 개념(트리, 그래프, BFS/DFS, 위상정렬)을 팀원 전원이 처음 학습하는 상황
- 코어타임을 "테스트"가 아닌 **"그날 배운 개념을 같이 확인하는 시간"**으로 운영
- 커리큘럼 안내: "문제를 풀지 못해도 괜찮습니다. 최선을 다한 흔적을 남겨보세요."

### 진행 방식 (매일 1시간)
1. **모의 풀이 (30분)**: 그날 학습 개념에 맞는 Easy~Medium 개념 확인 문제를 풀이
   - AI, 인터넷 사용 금지 (오프라인 환경)
   - 못 풀어도 OK — 의사코드, 접근법 메모 등 시도한 흔적 남기기
2. **개념 공유 (30분)**: 서로의 접근법과 개념 이해를 공유
   - "정답 맞추기"가 아니라 "왜 이 방법을 선택했는지" 중심으로 토의
   - 개념이 잘 잡힌 팀원이 설명해주고, 설명하면서 자기 이해도도 점검

### 문제 선정 기준
- 커리큘럼 코테 문제(Maximum Depth, Average of Levels, Number of Islands, Snakes and Ladders) 대신, 그날 학습 개념에 맞는 **개념 확인용 Easy 문제** 우선 선정
- 팀원별 진도 차이가 있으면 같은 주제의 다른 난이도 문제를 배정 가능
- 최소 3문제 이상 (4일간 진행)
- 요일별 추천 문제는 `코어타임 개념 확인 문제 리스트` 참고

## 코드 관리
- 각자 개인 레포지토리에서 작업
- 매일 풀이한 문제를 커밋·푸시
- 커밋 컨벤션: `알고리즘 커밋 컨벤션` 문서 참고

## 진척도 관리
- GitHub Projects 칸반보드 운영 (To Do → In Progress → Done)
- 이슈 템플릿으로 주차별 Task 등록
- 매일 스크럼 때 보드 상태 간단히 확인

## 팀 내 도움 요청 기준
- 혼자 30분 이상 같은 곳에서 막히면 팀원에게 먼저 질문
- 질문할 때는 "어디까지 시도했고 어디서 막혔는지"를 함께 공유
- 서로 가르쳐주는 과정 자체가 학습 — 설명하는 쪽도 이해도가 올라감

## 주간 회고 (목요일 마지막 코어타임)
- 핵심 역량 목표 달성률 팀원 간 점검
- 키워드 기반 모의 면접 실시
- 다음 주 개선할 점 공유
