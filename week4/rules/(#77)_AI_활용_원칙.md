# 금주차 AI 활용 원칙

## 팀 상황
- 이번 주 개념(트리, 그래프, BFS/DFS, 위상정렬)은 팀원 전원이 처음 학습

## 채택 수준: 🟢 학습 보조 수준 (강력 추천)
> 코드 구현을 제외하고는 자유롭게 사용 가능

---

## 허용 범위

### ✅ 적극 활용
- **개념 학습**: 책/자료로 1차 학습 후, 이해 안 되는 부분을 AI에 질문
- **문제 이해**: 문제 지문 해석, 입출력 예시 분석
- **힌트 요청**: "이 문제에 어떤 자료구조가 적합할까?" 수준의 방향성 질문
- **코드 리뷰**: 직접 작성한 코드의 시간복잡도 분석, 개선점 피드백
- **오류 분석**: 런타임 에러, 틀렸습니다 등의 원인 파악 보조

### ⚠️ 주의하며 사용
- **디버깅**: 에러 메시지를 보여주되, 수정 코드를 바로 받지 않고 원인만 파악
- **풀이 검증**: 내가 푼 후에 다른 접근법이 있는지 확인하는 용도

### ❌ 사용하지 않음
- **코드 구현 생성**: AI에게 풀이 코드를 작성하게 하는 것
- **과제 해답 생성**: 문제를 붙여넣고 정답을 받는 것
- **코어타임 코딩테스트**: AI, 인터넷 사용 제한 (커리큘럼 규칙)

---

## 팀 내 유연 조정 기준
- 베이직/하 문제: 최대한 AI 없이 시도 → 30분 이상 막히면 힌트 수준까지 허용
- 중 문제 (실버): AI 없이 시도 → 1시간 이상 막히면 접근법 힌트까지 허용
- 중 문제 (골드): AI 없이 시도 → 1.5시간 이상 막히면 힌트 + 유사 문제 풀이 참고 허용
- 팀원별 난이도 차이가 있으므로, 위 기준은 개인 상황에 맞게 조정 가능

## AI 사용 시 원칙
1. **질문은 구체적으로**: "이거 어떻게 풀어?"가 아니라 "BFS로 접근했는데 방문 체크 시점이 맞는지 모르겠다" 수준으로
2. **답변은 바로 수용하지 않기**: AI 답변을 받으면 "왜 그런지" 한 번 더 생각하고 적용
3. **코드는 직접 작성**: AI가 설명해준 로직을 보고 직접 코드로 옮기기 (복붙 금지)
